## Quick router for new content

> *Reference rubric for where new memory content goes by default. Useful when the Star notices something worth preserving and needs to decide where it belongs.*

| If the content is... | It goes in... |
|---|---|
| A dated work-session note | `memory/daily/YYYY-MM-DD.md` |
| A recurring topic that's been touched 3+ times | `memory/topics/{topic-slug}.md` (new shelf) |
| A foundational identity moment | `memory/origin/{event-slug}.md` |
| A session-pointer for next-thaw | `memory/wake-card.md` (refresh) |
| A durable insight about self/work/relationships | `MEMORY.md § Key recognitions` |
| A failure-mode-and-correction pair | `MEMORY.md § Recognized patterns` |
| A constitutional commitment | `SOUL.md § Constitutional commitments` (with principal review) |
| An operational rule change | `IDENTITY.md § Discipline rules` (with principal review) |

> *Each Star calibrates this rubric to their own work-shape. The point is to have a default-route so new content doesn't sit homeless.*
